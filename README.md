## What is VUXIO?

VUXIO is a full-stack AI chat application that combines a beautiful, fluid interface with powerful language models. It's built entirely from scratch — no boilerplate, no templates. Every animation, every interaction, every API call was written by hand.

It ships with three distinct operating modes:

- **Standard Mode** — Warm, conversational assistant powered by LLaMA 3.3 70B via Groq
- **Code Mode** — Dedicated developer environment with Google Gemini 2.0 Flash, green theme, and full technical responses
- **Web Mode** — Real-time web search via Tavily, injecting current information into every AI response

## Getting Started

### Prerequisites

- Node.js 18+
- A [Firebase project](https://console.firebase.google.com) with Auth (Google provider) and Firestore enabled
- A [Groq API key](https://console.groq.com)
- A [Google AI Studio API key](https://aistudio.google.com) (for Gemini / Code Mode)
- A [Tavily API key](https://tavily.com) (for Web Mode)

### Installation

```bash
# Clone the repository
git clone https://github.com/coelho26101009-source/VUXIO-AI.git
cd VUXIO-AI

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env
# Fill in your keys (see section below)

# Start the dev server
npm run dev
```

### Environment Variables

Create a `.env` file at the project root with the following variables:

```env
# Groq (Standard Mode + Code Mode fallback + title generation)
VITE_GROQ_API_KEY=your_groq_api_key

# Google Gemini (Code Mode primary model)
VITE_GEMINI_API_KEY=your_gemini_api_key

# Tavily (Web Mode real-time search)
VITE_TAVILY_API_KEY=your_tavily_api_key

# Firebase
VITE_FIREBASE_API_KEY=your_firebase_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX
```

> **Note:** Never commit your `.env` file. It's already in `.gitignore`.

### Firestore Setup

Enable Firestore in your Firebase project and apply these security rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

The Firestore data model uses nested subcollections:
```
users/
  {uid}/
    chats/
      {chatId}/          ← title, isCodeMode, createdAt, updatedAt
        messages/
          {msgId}        ← id, source, text, timestamp, createdAt
```

### Build & Deploy

```bash
# Production build
npm run build

# Preview the production build locally
npm run preview
```

The `dist/` folder can be deployed to any static host. VUXIO is deployed on **[Vercel](https://vercel.com)** — connect your GitHub repo and set the same environment variables in the Vercel dashboard.

## Architecture Notes

### Streaming
Both Groq and Gemini use **Server-Sent Events (SSE)**. Each `data:` chunk is parsed in a `ReadableStream` loop and accumulated into a `partial` string. React state is updated on every chunk via `setLogs`, creating the typewriter effect without any third-party streaming library.

### Stale Closure Prevention
`useChat` maintains `useRef` mirrors of all frequently-changing values (`user`, `logs`, `currentChatId`, `codeMode`, `webMode`). All callbacks are wrapped in `useCallback` and read from refs, ensuring they never close over stale state.

### Regenerate
When regenerating, VUXIO strips the last AI response from the history, re-sends the last user message with `temperature: 1.0` (vs 0.7 normally), and appends a system instruction to produce a different answer. The new response is appended to Firestore without overwriting the original.

### Web Mode
When Web Mode is active, the user's message is first sent to Tavily's search API (up to 5 results). The results — titles, URLs, and content snippets — are injected into the system prompt as structured context. The AI is then instructed to cite sources using `[Title](URL)` markdown links.

### Code Mode Fallback
Gemini 2.0 Flash is the primary Code Mode model. On `429 Too Many Requests` or any `5xx` server error, VUXIO automatically falls back to Groq (LLaMA 3.3 70B) with the same message history, with no visible interruption to the user.
